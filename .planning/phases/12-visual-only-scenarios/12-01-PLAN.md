<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| generator → schema validator | VisualOnlyGenerator produces dicts that validate_alert() must accept |
| seed → RNG stream | Seed isolation ensures visual-only RNG does not advance the MetadataGenerator stream |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-12-01 | Information Disclosure | visual_data.uri field | accept | Synthetic URIs (ucf-crime/test/{cat}/{i}.mp4) contain the UCF category name, which encodes GT. This is intentional benchmark structure — the URI is not sent to evaluated systems; only _meta is stripped. Evaluated systems receive visual_data.uri to fetch actual video frames, not to pattern-match the path. |
| T-12-02 | Tampering | RNG stream coupling | mitigate | VisualOnlyGenerator owns self.rng = np.random.RandomState(seed) exclusively. Never pass this rng to MetadataGenerator or share across generators. Test determinism (seed=42 twice) catches accidental sharing. |
| T-12-03 | Information Disclosure | Leakage via zone/device/context distributions | mitigate | Use identical sample_zone(), sample_device(), sample_site_type() shared distributions as MetadataGenerator. Leakage stump tests in test_visual_only.py confirm no field exceeds 70% accuracy. Run immediately after first generation. |
</threat_model>
