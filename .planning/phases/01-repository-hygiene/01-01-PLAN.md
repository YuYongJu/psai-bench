<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| filesystem → git index | `git rm --cached` removes from tracking but leaves disk files intact — risk of accidental `git rm` (without --cached) deleting working data |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-01-01 | Tampering | `git rm --cached` command | mitigate | Use `--cached` flag explicitly on every `git rm` call; verify disk files survive via `ls data/generated/` after |
| T-01-02 | Information Disclosure | Generated JSON in git history | accept (REPO-03 handles) | History purge deferred to Plan 02 — this plan only removes from HEAD tracking |
| T-01-03 | Tampering | ruff --unsafe-fixes | accept | F841 fix (remove unused variable assignment) is safe — the variable `both_right` is never read after assignment |
</threat_model>
