<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| local git repo → public remote | History rewrite is safe locally but catastrophic if pushed after clones exist |
| git filter-repo → working tree | filter-repo rewrites history only; untracked working tree files are unaffected |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-02-01 | Tampering | `git filter-repo --force` | mitigate | Use `--path data/generated/ --invert-paths` to strip only the target path; verify all other paths intact post-rewrite; human checkpoint before public push |
| T-02-02 | Denial of Service | Rewrite destroys source code | mitigate | Verify `ls psai_bench/` and `pytest` pass after rewrite before approving; checkpoint gate is blocking |
| T-02-03 | Information Disclosure | Old SHAs in documentation/STATE.md | accept | Stale SHA references are cosmetic; they do not expose data and will be updated in Phase 5 documentation work |
| T-02-04 | Tampering | pip install git-filter-repo from untrusted source | accept | git-filter-repo is a well-maintained official Git project (github.com/newren/git-filter-repo); pip install from PyPI is standard practice |
</threat_model>
