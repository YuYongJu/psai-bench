Class design:

```python
class ContradictoryGenerator:
    """Generate Visual-Contradictory Track scenarios where metadata misrepresents video content.

    Two sub-types:
    - Overreach: metadata signals suggest THREAT, video content is BENIGN (Normal category)
    - Underreach: metadata signals suggest BENIGN, video content is THREAT (anomaly category)

    GT always follows video content. metadata_derived_gt is stored in _meta for analysis
    but is never used as the final ground_truth.

    RNG isolation: owns its own np.random.RandomState (Pitfall 4).
    """

    def __init__(self, seed: int = 42):
        self.rng = np.random.RandomState(seed)
        self.seed = seed
```

Implement generate(n: int = 500) -> list[dict]:

1. Import at top of method (lazy import inside method, like existing generators):
   - CONTRADICTORY_THREAT_DESCRIPTIONS, CONTRADICTORY_BENIGN_DESCRIPTIONS from distributions
   - assign_ground_truth_v2 from distributions

2. Sub-type selection: ~50% overreach, ~50% underreach using rng.random() < 0.50.

3. For OVERREACH (video=BENIGN, metadata claims THREAT):
   - Sample category from BENIGN categories only: ["Normal"]
     (Normal is the only BENIGN category in UCF_CATEGORY_MAP)
   - video_derived_gt = "BENIGN" (from UCF_CATEGORY_MAP["Normal"]["ground_truth"])
   - Select metadata signals biased toward THREAT:
     - zone_type: rng.choice(["restricted", "utility"])
     - zone_sensitivity: rng.randint(4, 6) clipped to 5
     - time_of_day: rng.choice(["night", "dawn"])
     - severity: rng.choice(["HIGH", "CRITICAL"])
     - badge_minutes_ago: None (no badge softens the threat signal)
     - device fpr: sampled from low_quality profile (high FPR score, but zone+time dominate)
   - description: sampled from CONTRADICTORY_THREAT_DESCRIPTIONS

4. For UNDERREACH (video=THREAT, metadata claims BENIGN):
   - Sample category from THREAT categories only (exclude Normal, Arrest, RoadAccidents, Shoplifting):
     ["Abuse", "Arson", "Assault", "Burglary", "Explosion", "Fighting", "Robbery", "Shooting", "Stealing", "Vandalism"]
   - video_derived_gt = UCF_CATEGORY_MAP[cat]["ground_truth"] — always "THREAT" for these categories
   - Select metadata signals biased toward BENIGN:
     - zone_type: rng.choice(["parking", "interior"])
     - zone_sensitivity: rng.randint(1, 3)
     - time_of_day: "day"
     - severity: "LOW"
     - badge_minutes_ago: int(rng.randint(1, 10))  — recent badge strongly suppresses threat score
     - device fpr: sampled from high_quality profile (lower score)
   - description: sampled from CONTRADICTORY_BENIGN_DESCRIPTIONS

5. After selecting metadata signals, compute metadata_derived_gt by calling assign_ground_truth_v2
   with the biased signals. If metadata_derived_gt == video_derived_gt (the signals accidentally
   agree), resample the zone and time (up to 10 retries). After 10 failed retries, skip this
   scenario (subtract from count and continue) — do not generate an incoherent scenario.

6. Use the shared distribution helpers for the remaining fields:
   - zone object: {"id": f"zone-{rng.randint(100,999)}", "name": ..., "type": zone_type, "sensitivity": zone_sensitivity, "operating_hours": "24/7"}
   - device: sample_device(zone_type, rng)  — override device.false_positive_rate after sampling
   - weather: sample_weather(time_of_day, rng)
   - site_type: _sample_valid_site(cat, rng)

7. Build alert dict:
   - alert_id: f"ucf-contradictory-{i:05d}"
   - track: "visual_contradictory"
   - severity: the biased severity value (present and misleading — unlike visual_only which omits it)
   - description: from the contradictory pool (present and misleading)
   - visual_data: synthetic URI f"ucf-crime/test/{cat}/{i:05d}.mp4", type="video_clip",
     duration_sec, resolution, keyframe_uris=[]
   - _meta fields:
     - ground_truth: video_derived_gt  (always the video label)
     - video_derived_gt: video_derived_gt
     - metadata_derived_gt: metadata_derived_gt
     - contradictory: True
     - visual_gt_source: "video_category"
     - difficulty: _assign_difficulty(cat, zone_sensitivity, time_of_day, device fpr, rng)
     - source_dataset: "ucf_crime"
     - source_category: cat
     - seed: self.seed
     - index: i
     - generation_version: "v3"
     - adversarial: False
     - ambiguity_flag: False

Note on ZONE object construction: Use sample_zone(rng) then override type and sensitivity,
OR construct directly. Either is fine — just be consistent and avoid double-consuming RNG.
The key is to pass zone["type"] and zone["sensitivity"] (the biased values) to assign_ground_truth_v2.

Note on SUSPICIOUS categories: The THREAT pool for underreach only includes THREAT-labeled UCF
categories (never SUSPICIOUS). This ensures video_derived_gt is always "THREAT" for underreach
and "BENIGN" for overreach — making the divergence test deterministic.
