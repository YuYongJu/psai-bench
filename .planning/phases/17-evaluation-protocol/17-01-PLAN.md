## 1. Overview
- State the four evaluation tracks: metadata, visual_only, visual_contradictory, temporal
- Explain the core research question: does video perception add value over metadata-only triage
- State that GT derivation rules differ per track (forward to sections below)
- Note that _meta fields are not sent to evaluated systems — they are benchmark-side only

## 2. Track 1: Metadata Track
- GT derivation: five-signal weighted sum via assign_ground_truth_v2() in distributions.py
- Cross-reference docs/decision-rubric.md for the full signal scoring tables
  (do NOT duplicate the tables — reference the rubric document instead)
- State which fields contain the five signals: zone.type, zone.sensitivity,
  context.time_of_day, device.false_positive_rate, severity,
  context.recent_badge_access_1h
- Note that description text is NOT an input to GT derivation
- Scoring function: score_run(scenarios, outputs) → ScoreReport
- Worked example: one THREAT and one BENIGN case using the same format as decision-rubric.md
  Section 6 (full signal-by-signal table, total, threshold comparison)

## 3. Track 2: Visual-Only Track
- GT derivation: UCF_CATEGORY_MAP[category]["ground_truth"] directly — no weighted sum
- _meta.visual_gt_source == "video_category" on every scenario
- The evaluated system receives visual_data.uri (synthetic path format:
  "ucf-crime/test/{category}/{index}.mp4") — no severity or description fields
- Metadata fields present for context (zone, device, time_of_day, weather, site_type)
  but NOT used for GT assignment
- Leakage constraint: description and severity are intentionally absent so that
  no single text field reveals GT
- Scoring function: score_run(scenarios, outputs) → ScoreReport (same as metadata)
- UCF Crime → GT mapping table (extract from UCF_CATEGORY_MAP — show category, ground_truth
  for all 14 categories)
- Worked example: show a scenario dict snippet with the key fields, explain how GT is determined

## 4. Track 3: Visual-Contradictory Track
- GT derivation: GT always equals _meta.video_derived_gt, never _meta.metadata_derived_gt
- _meta.contradictory == True on every scenario
- Two sub-types:
  - Overreach: metadata signals → THREAT, video content → BENIGN (Normal category)
    Context bias: restricted/utility zone, night/dawn, HIGH/CRITICAL severity, low-FPR device
  - Underreach: metadata signals → BENIGN, video content → THREAT (anomaly category)
    Context bias: parking/interior zone, daytime, LOW severity, recent badge scan, high-FPR device
- Guarantee: metadata_derived_gt != video_derived_gt (enforced by generator retry loop)
- Scoring function: score_run(scenarios, outputs) → ScoreReport
  (GT = video_derived_gt, so a correct system ignores the misleading metadata)
- Worked example: show one overreach and one underreach scenario with _meta fields,
  explain which GT wins and why

## 5. Track 4: Temporal Track
- GT derivation: per-alert GT computed by assign_ground_truth_v2() applied to each alert's
  individual signals — same function as metadata track, but signals vary across the sequence
- Three escalation pattern types (stored in _meta.escalation_pattern):
  - monotonic_escalation: alerts before turn_point use LOW/MEDIUM severity + day zone,
    alerts at/after turn_point use HIGH severity + restricted zone + night
  - escalation_then_resolution: escalates to peak at turn_point, then badge scan clears post-turn alerts
  - false_alarm: first alert is HIGH/CRITICAL + restricted + night, subsequent alerts are LOW + parking + day + badge
- Sequence structure: _meta.sequence_id groups alerts, _meta.sequence_position is 1-based monotonic,
  _meta.sequence_length is the total count (3-5)
- Scoring function: score_sequences(scenarios, outputs) → SequenceScoreReport
  (NOT score_run — temporal sequences are evaluated as units, not individual alerts)
- SequenceScoreReport fields (list all with definitions):
  - n_sequences: total sequences evaluated
  - n_threat_sequences: sequences with at least one THREAT GT alert
  - n_benign_sequences: sequences where all GTs are BENIGN or SUSPICIOUS
  - early_detection_rate: fraction of threat sequences where model first returned THREAT at position 1 or 2
  - late_detection_rate: fraction of threat sequences where model first returned THREAT only at the last position
  - missed_sequence_rate: fraction of threat sequences where model never returned THREAT
  - false_escalation_rate: fraction of benign sequences where model returned THREAT on any alert
- Worked example: show a 4-alert monotonic_escalation sequence with _meta fields for each alert,
  and two example model output vectors (one early detect, one miss)

## 6. Scoring Functions Reference
- score_run(scenarios, outputs) — individual-alert scoring for metadata, visual_only, visual_contradictory
  - Input: list of scenario dicts + list of output dicts {alert_id, verdict, confidence}
  - Output: ScoreReport with TDR, FASR, Decisiveness, ECE, aggregate_score
  - Ambiguous partition: scenarios with _meta.ambiguity_flag=True are scored separately in
    ScoreReport.ambiguous_report (excluded from main aggregate)
  - Missing outputs: scored as incorrect with confidence=0 (cannot game by skipping)
- score_sequences(scenarios, outputs) — sequence-level scoring for temporal track
  - Input: same format as score_run; scenarios without sequence_id in _meta are silently skipped
  - Output: SequenceScoreReport (see Section 5)
- Aggregate score formula: 0.4*TDR + 0.3*FASR + 0.2*Decisiveness + 0.1*(1-ECE)
- Mixed-track files: use partition_by_track(scenarios) to split, then score each track separately,
  then call format_dashboard(report, track_reports=...) for combined display

## 7. Frame Extraction Baseline
- Purpose: establish the visual-only baseline — what performance does uniform frame sampling achieve?
- Function: extract_keyframes(video_path, keyframe_interval_sec=5.0, max_frames=50)
  from psai_bench.frame_extraction
- CRITICAL constraint (FRAME-02): selection is purely uniform interval. MUST NOT use
  _meta.anomaly_segments for frame selection. The evaluated system and the baseline
  both see the same frames regardless of whether anomaly annotations exist.
- Install: pip install "psai-bench[visual]" (opencv-python-headless optional dep)
- Returns: list of JPEG bytes — callers encode to base64 if needed for API payloads
- Parameters: keyframe_interval_sec controls density, max_frames caps memory use on long videos
- Determinism: given the same video file and parameters, output is identical across runs
- Typical workflow: call extract_keyframes() → encode frames → send to model API → collect verdict

## 8. Cross-Track Gap Analysis (Perception-Reasoning Gap)
- Formula: gap = metadata_aggregate_score - visual_aggregate_score
  (implemented in compute_perception_gap(metadata_report, visual_report))
- Interpretation:
  - gap > +0.02: metadata context helps — video perception alone is insufficient for this model
  - gap < -0.02: visual track outperforms — model benefits from seeing video
  - |gap| < 0.02: negligible difference
- CLI: psai-bench analyze-frame-gap --metadata-results ... --visual-results ...
  --metadata-scenarios ... --visual-scenarios ...
  (loads both files, calls score_run() on each, then compute_perception_gap())
- Note: use visual_only or visual_contradictory track for the visual side — not the legacy
  "visual" track (which has metadata + video)
- Dashboard preview: format_dashboard(report, track_reports={"metadata": ..., "visual_only": ...})
  renders a gap preview inline when both tracks are present

## 9. CLI Quick Reference
- Generate:
  - psai-bench generate --track metadata --source ucf --n 3000 --seed 42
  - psai-bench generate --track visual_only --n 500 --seed 42
  - psai-bench generate --track visual_contradictory --n 500 --seed 42
  - psai-bench generate --track temporal --n 50 --seed 42  (n = number of sequences)
- Score:
  - psai-bench score --scenarios scenarios.json --outputs outputs.json
  - psai-bench score-sequences --scenarios temporal.json --outputs outputs.json
- Gap:
  - psai-bench analyze-frame-gap --metadata-results ... --visual-results ...
    --metadata-scenarios ... --visual-scenarios ...
