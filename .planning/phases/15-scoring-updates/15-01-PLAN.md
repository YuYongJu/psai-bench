  <behavior>
    score_sequences() groups by _meta.sequence_id, classifies each sequence as threat or benign,
    then computes:

    - early_detection_rate: fraction of threat sequences where model escalated to THREAT within
      the first 2 alerts of the sequence overall (sequence positions 1-2, i.e. 0-indexed 0 or 1)
    - late_detection_rate: fraction of threat sequences where model first reached THREAT only at
      the last alert position (0-indexed: index == len(sequence) - 1)
    - missed_sequence_rate: fraction of threat sequences where model never returned "THREAT"
      on any alert in the sequence
    - false_escalation_rate: fraction of benign sequences where model returned "THREAT" on
      any alert in the sequence

    A sequence is "threat" if any alert in it has ground_truth == "THREAT".
    A sequence is "benign" if all alerts have ground_truth == "BENIGN" or "SUSPICIOUS".

    Alerts without sequence_id in _meta are silently skipped (mixed files allowed).

    Test cases (hand-computable):
    - 4-alert threat sequence (turn_point=3): positions 1-2 GT=BENIGN/SUSPICIOUS, positions 3-4 GT=THREAT.
      If model outputs THREAT at position 1 or 2: early=1, late=0, missed=0
      If model outputs THREAT at position 3: early=0, late=0, missed=0 (detected, but not early or late)
      If model outputs THREAT only at position 4 (last): early=0, late=1, missed=0
      If model never outputs THREAT: early=0, late=0, missed=1
    - 3-alert benign sequence (false_alarm): position 1 GT=THREAT, positions 2-3 GT=BENIGN.
      Classify as threat sequence (has THREAT GT). Model never returns THREAT: missed=1.
    - pure benign sequence (all GT=BENIGN): model returns THREAT on alert 2: false_escalation_rate=1.
  </behavior>

2. Add score_sequences() function (insert after SequenceScoreReport, before score_run()):

```python
def score_sequences(
    scenarios: list[dict],
    outputs: list[dict],
) -> SequenceScoreReport:
    """Score temporal sequence evaluation.

    Groups alerts by _meta.sequence_id, sorts by _meta.sequence_position,
    evaluates each sequence as a unit.

    Alerts without sequence_id in _meta are silently ignored — mixed files allowed.
    Does NOT call or modify score_run().
    """
```

Implementation logic:
- Build output_map = {o["alert_id"]: o for o in outputs}
- Group scenarios by s["_meta"].get("sequence_id") — skip None/missing
- For each group, sort by s["_meta"]["sequence_position"]
- Classify group: if any s["_meta"]["ground_truth"] == "THREAT" -> threat_sequence, else benign_sequence
- For threat sequences:
  - early: model verdict == "THREAT" appears at index 0 or 1 (first two alerts, 0-indexed from start of sequence)
  - late: first THREAT verdict is at the last alert (index == len(sorted_alerts) - 1)
  - missed: no alert in sequence has model verdict == "THREAT"
- For benign sequences:
  - false_escalation: any alert in sequence has model verdict == "THREAT"
- Store per_sequence_results[seq_id] = {"pattern": escalation_pattern, "is_threat_seq": bool, "model_verdicts": list}
- Return SequenceScoreReport with all rates computed

def partition_by_track(scenarios: list[dict]) -> dict[str, list[dict]]:
    """Partition scenarios by their track field. Returns dict[track_name, scenarios]."""
    partitions: dict[str, list[dict]] = {}
    for s in scenarios:

        partitions.setdefault(track, []).append(s)
    return partitions
