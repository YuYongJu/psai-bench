<objective>
Audit the scenario generator for structural site-type leakage: verify that a logistic regression probe trained on non-site features cannot predict site_type above chance.

Purpose: SITE_CATEGORY_BLOCKLIST (generators.py line 51) creates a category-site correlation — solar scenarios never contain Shoplifting/Robbery. This could let a model infer site identity from the category distribution without ever seeing context.site_type, which would compromise the generalization test's validity. The audit must confirm the leakage is below 60% accuracy before the multi-site generalization metric is trusted.

Output: tests/test_site_leakage.py — a repeatable pytest test that runs as part of the standard suite and will catch future leakage regressions if distributions.py changes.
</objective>

Probe:
```python
from sklearn.linear_model import LogisticRegression
from sklearn.model_selection import cross_val_score

# 5-fold stratified cross-validation avoids small-set overfitting
scores = cross_val_score(
    LogisticRegression(max_iter=500, random_state=0),
    X,  # feature matrix, no site_type columns
    y,  # site_type labels (encoded as integers)
    cv=5,
    scoring="accuracy",
)
mean_accuracy = scores.mean()
```

Threshold assertion:
```python
LEAKAGE_THRESHOLD = 0.60

    f"Check SITE_CATEGORY_BLOCKLIST in generators.py — categories may uniquely identify sites."
)
```
